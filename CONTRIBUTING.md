# Contributing to Slopocalypse

Thank you for your interest in contributing to Slopocalypse! This document provides guidelines and instructions for contributing.

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Install dependencies: `npm install`
4. Create a new branch for your changes

## Development

```bash
# Start the dev server
npm run dev

# Run the backfill script
npm run backfill
```

## Making Changes

1. Create a feature branch from `main`
2. Make your changes
3. Test your changes locally
4. Commit with clear, descriptive messages
5. Push to your fork and open a pull request

## Pull Request Guidelines

- Provide a clear description of the changes
- Link any related issues
- Ensure the server starts and runs without errors

## Reporting Issues

Use [GitHub Issues](https://github.com/sestinj/slopocalypse/issues) to report bugs or request features. Please use the provided issue templates.

## Code of Conduct

Be respectful and constructive in all interactions. We are committed to providing a welcoming and inclusive experience for everyone.
